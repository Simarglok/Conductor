# Conductor — open-source data transformation platform
#
# Repo:  https://github.com/Simarglok/Conductor
# Root:  /Users/simarglok/Git/Conductor
# Stack: dbt-core 2.0.0a4 · Airflow 3.3.0 · FastAPI · PostgreSQL 18 · Redis 8.6
# Host:  macOS (Apple Silicon) · Docker Desktop
#
# ── Quick commands ──
# Start stack:        docker compose --profile init up airflow-db-init && docker compose up -d
# Stop stack:         docker compose down
# Full wipe:          docker compose down -v
# Rebuild:            docker compose build
# Logs:               docker compose logs -f <service>
# Init DB (first):    docker compose --profile init up airflow-db-init
#
# ── Services ──
# FastAPI (Conductor)    http://localhost:8000/docs
# Airflow UI             http://localhost:8080  (admin / admin)
# code-server (VS Code)  http://localhost:8443  (password: conductor)
# PostgreSQL             localhost:5432  (conductor / conductor)
# Redis                  localhost:6379
#
# ── Dir structure ──
# backend/        — FastAPI control plane (Python 3.14, SQLAlchemy async, Alembic)
# dags/           — Airflow DAGs + dbt projects
# docker/         — Dockerfiles (airflow/, code-server/)
# docker-compose.yml — full dev environment
# workspaces/     — code-server user data (gitignored)
# logs/           — Airflow task logs (gitignored)
#
# ── Config ──
# Secrets are in .env (gitignored) at project root.
# Backend config is ENV-only via CONDUCTOR_* variables.
# Airflow config is ENV-only (no airflow.cfg).
#
# ── Python (local dev) ──
# python3.14 -m venv .venv
# env -u PYTHONPATH .venv/bin/pip install -e "backend[dev]"
